## 项目概述

这是一个**营销素材拆解与裂变系统**的设计文档与原型项目。面向抖音效果广告投放业务，核心目标是将优质营销素材拆分为可复用的骨架，通过骨架+新内容组合批量产出新素材。

项目目前处于 **设计文档 + HTML 原型阶段**，尚未进入正式开发。

## 核心概念

### 五层拆解模型（L1-L5）

每个素材从抽象到具体拆解为 5 层：

| 层级 | 名称 | 属性 | 说明 |
|------|------|------|------|
| L1 | 主题层 | 枝干（可复用） | 素材讲什么 |
| L2 | 策略层 | 枝干（可复用） | 用什么策略/情绪打动用户 |
| L3 | 结构层 | 枝干（可复用） | 内容骨架/段落逻辑 |
| L4 | 元素层 | 枝杈（可插拔） | 标题公式、钩子句式、过渡方式等 |
| L5 | 表达层 | 叶子（可替换） | 具体文字/视觉表达 |

### 三种裂变模式

- **换叶子（replace_leaf）**: L1+L5 替换，骨架不变 — 最可靠，效果保留约 85%
- **换枝杈（replace_branch）**: L3+L4 替换，主题不变 — 效果保留约 65%
- **换表达（replace_style）**: L2+L5 替换，骨架不变 — 效果保留约 70%

### 系统数据流

```
优质素材 → 拆解引擎(L1-L5) → 骨架库 + 元素库 → 裂变引擎(骨架×新元素) → 新素材 → 投放验证 → 结果回写
```

## 技术方案

### 推荐技术栈

- **前端**: HTML + CSS + JavaScript（原型阶段无框架，后续可渐进式引入 Vue 3）
- **后端**: Node.js + Express（推荐）或 Python + FastAPI
- **数据库**: MySQL 8.0（JSON 字段支持 L3/L4 结构化数据）
- **MVP 最简方案**: Python Flask + SQLite，1-2 天跑通核心流程

### 数据库核心表（6 张）

1. **material** — 素材表（原始内容、平台、品类、状态）
2. **dismantle** — 拆解表（L1-L5 结构化数据，L2/L3/L4/L5 用 JSON 字段）
3. **skeleton** — 骨架表（可复用的 L2-L4，含效果统计 avg_roi/avg_ctr）
4. **fission** — 裂变表（裂变配置、产出内容、预测/实际效果）
5. **effect_data** — 效果数据表（投放数据回写：CTR、ROI、CVR 等）
6. **tag / material_tag** — 标签表 + 素材标签关联表

### 核心 API

```
素材管理:   POST/GET /api/material
拆解引擎:   POST/GET /api/dismantle
骨架库:     POST/GET /api/skeleton
裂变引擎:   POST/GET /api/fission
效果数据:   POST/GET /api/effect
```

详细的 SQL 建表语句、API 请求/响应格式、后端代码示例见 `技术.md`。

## 开发阶段

| 阶段 | 内容 | 目标 |
|------|------|------|
| **P1** | 人工填写 L1-L5 拆解模板，存成结构化数据 | 跑通拆解流程，积累骨架库 |
| **P2** | 裂变页面：选择骨架+输入新内容→自动组合输出 | 跑通裂变流程 |
| **P3** | AI辅助拆解 + 自动生成变体 | 提升效率 |

MVP 开发排期：1 人 1 周（含数据库、后端 API、前端 4 个页面、测试部署）。

## 系统架构要点

- **拆解引擎本质**: 人工填写 + 模板引导（P1 不依赖 AI）
- **裂变引擎本质**: 模板填充 — 骨架是模板，新内容是填充物
- **效果闭环**: 投放后回写 effect_data → 自动更新 skeleton 的 avg_roi/avg_ctr → 指导下次裂变选骨架
- **核心矛盾**: 离散度（拆得越细越灵活）vs 聚合度（管理成本），五层模型是平衡点
- **AI 边界**: P1/P2 完全不需要 AI；AI 是 P3 的加速器（自动识别结构、自动生成变体）
