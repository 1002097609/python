#### 2. 全面清理硬编码，实现全动态数据驱动（commit: e0b97e0）

**核心目标：系统从 demo 级向企业级演进，所有业务数据和配置改为数据库驱动，零硬编码。**

##### 后端变更（fission.py）

- **补全裂变引擎**：`replace_branch`（换枝杈）和 `replace_style`（换表达）模式完整实现
  - replace_branch：从 replacement.L3 读取新结构段落内容
  - replace_style：从 replacement.L2/L5 读取新策略和表达
- **预测参数动态化**：`_predict_performance()` 优先从数据库 option 表读取效果保留系数
- **预测范围修正**：改为对称 +/-10%（修复了之前 ROI 范围不对称的问题）
- **output_title 简化**：去除硬编码的 `" — 裂变产出"` 后缀

##### 种子数据变更（seed_options.py）

- **upsert 安全模式**：seed() 函数从 destructive delete+reinsert 改为 merge 模式
  - 按 (group_key, value) 匹配，存在则更新，不存在则插入
  - 用户通过 UI 添加的自定义选项不会被清除
- **新增分组**：
  - `fission_status`：草稿(0) / 待审核(1) / 已采用(2) / 已投放(3)
  - `material_status`：未拆解(0) / 已拆解(1) / 已归档(2)
- **fission_mode label 格式升级**：`"名称|图标|描述"`（如 `"换叶子|🍃|效果保留85%"`）
- **通用化模板数据**：
  - golden_sentence：16 条通用营销金句（去掉了护肤/母婴/零食领域特定内容）
  - data_ref：14 条通用数据引用模板
  - visual_desc：20 条通用视觉描述

##### 路由修复（options.py）

- 修复 `/api/options/` 的 307 重定向问题，兼容带/不带尾部斜杠

##### 前端变更

- **FissionRecords.vue**：状态筛选 / 统计卡片标签 / 裂变模式标签全部改为从数据库动态加载
- **MaterialLib.vue**：素材状态标签从数据库 material_status 分组动态获取
- **Fission.vue**：裂变模式图标和描述从数据库 fission_mode 分组动态解析
- **Dismantle.vue**：默认 L3 结构从硬编码三段式改为空数组
- **SkeletonLib.vue**："高复用"阈值从硬编码 >5 改为动态计算（超过平均使用率）
