## 一、这个系统最终要做什么？

### 一句话概括

**把优质营销素材"拆"成可复用的骨架，再用骨架"生"出大量新素材——让素材生产从"靠灵感、靠人堆"变成"靠系统、靠方法"。**

### 业务背景

这是一家整合营销公司，核心业务是**抖音效果广告投放**。广告投放的核心资产就是素材（短视频、图文等），但过去素材生产高度依赖个人经验：

- 好素材为什么好？说不清
- 好素材能不能复用？靠人肉copy
- 新素材怎么批量生产？堆人、堆时间

这套系统要解决的，就是这个**从"经验驱动"到"系统驱动"**的跨越。

## 二、素材如何拆解？

### 核心类比：树结构

```
            🌳 一棵完整的素材树
           ╱ │ ╲
          /  │  \
         ╱   │   ╲
        🍃  🍃  🍃    ← 叶子（L5 表达层）—— 具体的文字/画面
       ╱│╲  ╱│╲
      ╱ │ ╲╱ │ ╲      ← 枝杈（L4 元素层）—— 标题公式、钩子句式
     ╱  │  ╳  │  ╲
    ╱   │ ╱ ╲  │   ╲   ← 树枝（L2-L3 策略/结构层）—— 框架逻辑
   ═════╪═══════╪═════
        │       │      ← 树干（L1 主题层）—— 讲什么
       ═╧═══════╧═
```

**拆解的本质**：把一棵完整的树拆成 **骨架（树干+树枝）+ 叶子（具体表达）**，骨架可以复用，叶子可以替换。

### 五层拆解模型

每个素材可以拆解为 **5 个层级**，从抽象到具体：

```
抽象 ◀──────────────────────────────────────▶ 具体

  L1        L2        L3        L4        L5
主题层  →  策略层  →  结构层  →  元素层  →  表达层
  │         │         │         │         │
讲什么    怎么打    怎么搭    用什么    具体是
         动人心的  框架的    什么零件   什么内容

 ◄── 枝干（可复用）──►  ◄─ 枝杈 ─►  ◄叶子（可替换)►
```

| 层级 | 名称 | 定义 | 是否可复用 | 示例（小红书种草文） |
|------|------|------|-----------|---------------------|
| **L1** | 主题层 | 素材讲什么 | ✅ 可复用 | "秋冬保湿面霜推荐" |
| **L2** | 策略层 | 用什么策略/情绪打动用户 | ✅ 可复用 | "成分党视角 + 对比测评 + 个人踩坑经历" |
| **L3** | 结构层 | 内容骨架/段落逻辑 | ✅ 可复用 | "痛点开头→产品介绍→成分对比→使用感受→总结推荐" |
| **L4** | 元素层 | 关键组成单元 | ⚡ 可插拔 | 标题公式、封面构图、emoji用法、转折句式、互动引导语 |
| **L5** | 表达层 | 具体的文字/视觉表达 | ❌ 可替换 | 具体的每一句话、每个画面、每张图 |

### 拆解示例：一篇抖音带货短视频脚本

```
┌─────────────────────────────────────────────────┐
│              原始素材（脚本）                      │
│                                                 │
│  "姐妹们！冬天脸干到爆皮？我踩了8个坑才找到     │
│   这款面霜...先看成分表，烟酰胺排第3位！         │
│   对比我之前用的XX牌...上脸感受完全不一样...     │
│   用了一周，你看这个状态...评论区告诉我你的       │
│   肤质，我帮你选！"                               │
└───────────────────────┬─────────────────────────┘
                        │ 拆解
                        ▼
┌─────────────────────────────────────────────────┐
│  L1 主题层：秋冬保湿面霜推荐                      │
│  L2 策略层：成分党视角 + 对比测评 + 个人踩坑      │
│  L3 结构层：                                      │
│    ├─ 开头(15%)：痛点共鸣 → "脸干到爆皮？"       │
│    ├─ 主体(60%)：成分对比 + 使用感受              │
│    └─ 结尾(25%)：效果展示 + 互动引导             │
│  L4 元素层：                                      │
│    ├─ 标题公式：情绪词 + 数字 + 结果              │
│    ├─ 钩子句式："我踩了N个坑才找到..."           │
│    ├─ 转折方式："对比之前用的..."                 │
│    └─ 互动设计："评论区告诉我你的肤质"            │
│  L5 表达层：                                      │
│    ├─ 金句："烟酰胺排第3位！"                    │
│    ├─ 数据："8个坑"、"一周"                      │
│    └─ 视觉：成分表特写、上脸对比、一周后状态       │
└─────────────────────────────────────────────────┘
```

### 三种拆解输入方式

```
素材录入
├── 🅰️ 粘贴原始素材 ──▶ AI辅助自动拆解（P3阶段）
├── 🅱️ 手动逐层填写 ──▶ 模板引导拆解（P1阶段 MVP）
└── 🅲️ 批量导入 ──▶ JSON/CSV结构化数据（P2阶段）
```

### 拆解后的数据结构

```
素材库
├── 原始素材（完整内容）
├── 拆解结果（L1-L5结构化数据）
│     ├── L1: 主题
│     ├── L2: 策略标签
│     ├── L3: 结构段落（含功能、占比）
│     ├── L4: 元素模块（标题公式、钩子等）
│     └── L5: 具体表达（金句、数据、视觉）
└── 标签（平台、品类、风格等）

         │ 提取骨架
         ▼

骨架库（可复用资产）
├── 结构模板（L3 段落框架）
├── 元素模块（L4 标题公式、钩子句式等）
└── 风格库（L2 策略 + L5 调性特征）
```

## 三、如何基于现有素材做裂变？

### 核心逻辑

```
裂变 = 保留骨架 + 替换内容 = 长出新树

     🌳 母体素材                🌱 裂变新素材
     ╱ │ ╲                     ╱ │ ╲
    ╱  │  ╲      骨架保留     ╱  │  ╲
   🍃  🍃  🍃  ──────────▶  🌸  🌸  🌸  ← 新叶子
   具体表达（旧）               具体表达（新）
```

### 三种裂变模式

| 模式 | 操作 | 骨架变不变 | 变的是什么 | 举例 |
|------|------|-----------|-----------|------|
| **同构异内容**（换叶子） | 骨架不变 + 主题/品类替换 | ❌ 不变 | L1主题 + L5表达 | 面霜测评 → 防晒霜测评 |
| **同内容异结构**（换枝杈） | 主题不变 + 结构调整 | ⚠️ 微调 | L3结构 + L4元素 | 痛点开头 → 悬念开头 |
| **跨风格移植**（换表达） | 骨架不变 + 风格替换 | ❌ 不变 | L2策略 + L5表达 | 成分党风格 → 闺蜜聊天风格 |

### 裂变流程（5步）

```
Step 1              Step 2              Step 3
┌──────────┐    ┌──────────────┐    ┌──────────────┐
│ 选择母体  │───▶│  选择裂变模式  │───▶│  指定替换内容  │
│   素材    │    │              │    │              │
└──────────┘    │ ○ 换叶子     │    │ 新主题/新结构 │
                │ ○ 换枝杈     │    │ /新风格      │
                │ ○ 换表达     │    └──────┬───────┘
                └──────────────┘           │
                                           ▼
Step 5              Step 4
┌──────────┐    ┌──────────────────┐
│ 输出裂变  │◀───│ 系统提取骨架      │
│ 素材列表  │    │ + 融合新元素      │
└─────┬────┘    └──────────────────┘
      │
      ├──── 直接可用 ✅
      │
      └──── 人工微调审核 🔍
```

### 裂变矩阵：批量产出

当一个素材库有 N 个素材时，裂变空间为：

```
裂变数量 = N × 骨架复用数 × 风格变体数 × 主题替换数

┌────────────────────────────────────────────┐
│           裂变矩阵示意图                      │
│                                            │
│            风格A    风格B    风格C          │
│  骨架1  │  M1-A  │  M1-B  │  M1-C  │       │
│  骨架2  │  M2-A  │  M2-B  │  M2-C  │  × 5个新主题
│  骨架3  │  M3-A  │  M3-B  │  M3-C  │       │
│                                            │
│  3 × 3 = 9 种组合 × 5 = 45 个新素材       │
└────────────────────────────────────────────┘
```

**实际案例**：

```
10个优质素材 × 3种结构变体 × 3种风格变体 × 5个新主题 = 450个新素材

这450个素材可以直接用于:
├── A/B测试（验证哪种骨架+风格组合效果最好）
├── 多平台投放（抖音/小红书/快手各用不同风格）
└── 多品类扩展（同一骨架覆盖不同产品）
```

### 三种裂变模式详细展开

#### 模式一：同构异内容（最常用）

```
母体：面霜测评脚本
├── L1: 秋冬保湿面霜推荐        ──▶ 替换为：防晒霜推荐
├── L2: 成分党 + 对比测评        ──▶ 保持不变 ✅
├── L3: 痛点→成分→对比→感受→推荐 ──▶ 保持不变 ✅
├── L4: 标题公式"踩了N个坑"     ──▶ 保持不变 ✅
└── L5: "烟酰胺排第3位"         ──▶ 替换为："SPF50+"

产出：防晒霜测评脚本（骨架完全复用，只换具体产品信息和数据）
```

#### 模式二：同内容异结构

```
母体：面霜测评脚本（痛点开头）
├── L1: 秋冬保湿面霜推荐        ──▶ 保持不变 ✅
├── L2: 成分党 + 对比测评        ──▶ 保持不变 ✅
├── L3: 痛点开头 → 成分 → 对比   ──▶ 改为：悬念开头 → 揭秘 → 对比
├── L4: "脸干到爆皮？"           ──▶ 改为："你用的面霜可能在伤害你"
└── L5: 全部重写                ──▶ 新结构需要新的具体表达

产出：同一产品的新版本脚本（测试不同开头哪种CTR更高）
```

#### 模式三：跨风格移植

```
母体：成分党风格的面霜测评
├── L1: 秋冬保湿面霜推荐        ──▶ 保持不变 ✅
├── L2: 成分党视角              ──▶ 改为：闺蜜聊天风格
├── L3: 痛点→成分→对比→感受→推荐 ──▶ 基本不变 ✅
├── L4: 专业术语 + 数据引用      ──▶ 改为：口语化 + emoji
└── L5: "烟酰胺排第3位"         ──▶ 改为："姐妹们这个真的绝绝子！"

产出：同一产品的闺蜜种草风格脚本（适配不同受众群体）
```

## 七、关键认知总结

### 拆解的核心价值

> **让"好素材为什么好"从不可说变成可描述、可传递、可复用。**

一个投手做的爆款视频脚本，过去只有他自己能复制。拆解后，他的经验变成了骨架库里的资产，任何人都可以在这个骨架上产出新素材。

### 裂变的核心价值

> **从"1个素材靠1个人"变成"1个骨架生100个素材"。**

```
传统模式：   1个投手 × 1天 = 3-5个素材
裂变模式：   1个骨架 × 10个主题 × 3种风格 = 30个素材/天
```

### 系统飞轮效应

```
更多素材 ──▶ 更多拆解 ──▶ 更丰富的骨架库
    ▲                          │
    │                          ▼
更优投放 ◀── 更精准裂变 ◀── 更多裂变可能
    │
    ▼
更多数据 ──▶ 知道哪些骨架好 ──▶ 裂变更精准
```
