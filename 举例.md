## 裂变全景图

以上只是**一次裂变**。实际上，一个骨架可以持续裂变：

```
                    🌳 骨架 SKL001
                   "踩坑逆袭+对比带货"
                         │
          ┌──────────────┼──────────────────┐
          │              │                  │
          ▼              ▼                  ▼
    🔄 换主题       🔄 换结构          🔄 换风格
          │              │                  │
    ┌─────┤        ┌─────┤            ┌─────┤
    │     │        │     │            │     │
    ▼     ▼        ▼     ▼            ▼     ▼
  零食   母婴     悬念  倒叙         闺蜜  专家
  带货   用品     开头  揭秘         聊天  测评
    │     │        │     │            │     │
    ▼     ▼        ▼     ▼            ▼     ▼
  2条    2条      2条    2条         2条   2条

  仅这1个骨架 × 3种模式 × 每个模式2个变体 = 12条新素材
```

如果骨架库中有 **10个优质骨架**，每个骨架做类似裂变：

```
10个骨架 × 3个结构变体 × 3个风格变体 × 5个新主题 = 450条新素材
```

这450条素材可以：
- **直接投放** A/B测试，验证哪种组合效果最好
- **多平台分发** 抖音用成分党风格，小红书用闺蜜风格
- **持续迭代** 投放数据回写 → 标记哪些骨架+品类组合ROI最高 → 指导下次裂变方向
```

## 总结：拆解→裂变的核心心法

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   1️⃣ 拆：把"好素材"拆成"骨架 + 表达"                     │
│      → 骨架是方法，表达是内容                              │
│                                                          │
│   2️⃣ 存：骨架入库，成为可复用资产                          │
│      → 骨架库越丰富，裂变空间越大                          │
│                                                          │
│   3️⃣ 裂：骨架不动，替换内容                               │
│      → 换主题、换结构、换风格，三种模式自由组合             │
│                                                          │
│   4️⃣ 验：新素材投放，数据回写                             │
│      → 知道哪些骨架+内容组合效果最好                       │
│                                                          │
│   5️⃣ 优：用数据反馈优化骨架库                              │
│      → 好骨架多用，差骨架淘汰                              │
│                                                          │
│   最终效果：从"1个投手1天写3条"到"1个骨架1天生30条"       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
