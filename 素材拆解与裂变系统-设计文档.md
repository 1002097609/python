# 素材拆解与裂变系统 — 设计文档

> 版本：v1.0 | 日期：2026-04-30

---

## 一、核心概念

### 类比：树结构

```
素材 = 树干 + 树枝 + 叶子

- 树干/枝干 → 骨架（结构、框架、逻辑链路）—— 可复用
- 叶子 → 血肉（具体案例、数据、表达、风格）—— 可替换

拆解：把一棵完整的树拆成 骨架 + 叶子
裂变：保留骨架，换掉叶子，长出一棵新树
```

---

## 二、素材拆解 — 多层模型

每个素材可以拆解为 **5 个层级**，从抽象到具体：

| 层级 | 名称 | 说明 | 示例（以一篇小红书种草文为例） |
|------|------|------|-------------------------------|
| L1 | **主题层** | 素材讲什么 | "秋冬保湿面霜推荐" |
| L2 | **策略层** | 用什么策略/情绪打动用户 | "成分党视角 + 对比测评 + 个人踩坑经历" |
| L3 | **结构层** | 内容骨架/段落逻辑 | "痛点开头 → 产品介绍 → 成分对比 → 使用感受 → 总结推荐" |
| L4 | **元素层** | 关键组成单元 | 标题公式、封面构图、emoji用法、转折句式、互动引导语 |
| L5 | **表达层** | 具体的文字/视觉表达 | 具体的每一句话、每个画面、每张图 |

### 关键区分

- **L1-L3 是"枝干"** → 可复用的框架
- **L4 是"枝杈"** → 可插拔的模块
- **L5 是"叶子"** → 具体的内容填充

## 四、裂变机制

### 4.1 裂变的三种模式

| 模式 | 说明 | 举例 |
|------|------|------|
| **同构异内容**（换叶子） | 骨架不变 + 主题/品类替换 | 面霜测评骨架 → 换成防晒霜测评 |
| **同内容异结构**（换枝杈） | 主题不变 + 结构调整 | 痛点开头 → 改成悬念开头 |
| **跨风格移植**（换表达） | 骨架不变 + 风格/调性替换 | 成分党风格 → 改成闺蜜聊天风格 |

### 4.2 裂变流程

```
Step 1: 选择母体素材
         ↓
Step 2: 选择裂变模式（换叶子 / 换枝杈 / 换表达）
         ↓
Step 3: 指定替换内容（新主题 / 新结构 / 新风格）
         ↓
Step 4: 系统提取骨架 + 融合新元素
         ↓
Step 5: 输出裂变素材列表（可人工微调）
```

### 4.3 裂变矩阵

当一个素材库有 N 个素材时，裂变空间为：

```
裂变数量 = N × 骨架复用数 × 风格变体数 × 主题替换数

例：10个素材 × 3种结构 × 3种风格 × 5个新主题 = 450个新素材
```

## 六、最小可行方案（MVP）

### 建议分三个阶段推进：

| 阶段 | 内容 | 目标 |
|------|------|------|
| **P1** | 人工填写 L1-L5 拆解模板，存成结构化数据 | 跑通拆解流程，积累骨架库 |
| **P2** | 开发裂变页面：选择骨架 + 输入新内容 → 自动组合输出 | 跑通裂变流程 |
| **P3** | AI辅助拆解 + 自动生成变体 | 提升效率 |

### MVP 数据表结构

```
素材表：id | 原始内容 | 主题 | 标签 | 创建时间
拆解表：素材ID | L1 | L2 | L3 | L4 | L5
骨架表：骨架ID | 结构JSON | 来源素材ID | 使用次数
裂变表：裂变ID | 母体骨架ID | 替换内容 | 输出结果
```

## 七、后续迭代方向

- **标签体系 & 检索**：按平台/品类/风格/策略等多维检索素材和骨架
- **效果数据闭环**：关联素材的投放数据，标记哪些骨架/元素效果好，指导裂变方向
- **AI辅助拆解**：输入原始素材，自动识别并填充 L1-L5
- **批量裂变**：指定多个骨架 + 多个主题/风格，批量生成素材矩阵
