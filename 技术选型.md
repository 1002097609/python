# 技术选型决策

> 日期：2026-04-30
> 决策者：AI 工程师 + AI 产品经理

---

## 结论：方案 C（Python Flask + SQLite + 纯 HTML/JS）

### 一句话

**用你最熟的语言、最轻的量级、最快的速度，先跑通闭环。**

---

## 决策依据

| 考量因素 | 结论 |
|---------|------|
| **技术背景** | 主 Python →  Flask 学习成本最低，开发最快 |
| **阶段目标** | 快速验证 Demo → 不需要 MySQL/ORM/构建工具等重型基础设施 |
| **AI 接入** | Python 生态 → P3 阶段接 LangChain/OpenAI SDK 无缝衔接 |
| **迭代路径** | Flask → FastAPI → 加 MySQL，渐进式升级，不推翻重来 |
| **维护成本** | 一个人维护，SQLite 零配置，不需要 DBA |

---

## 技术栈详设

### 后端

| 选型 | 版本 | 说明 |
|------|------|------|
| **Python** | 3.11+ | 主语言 |
| **Flask** | 3.x | 轻量 Web 框架，几十行代码跑通 API |
| **SQLite** | 内置 | 零配置，一个文件就是整个数据库，Python 内置支持 |
| **SQLAlchemy** (可选) | 2.x | 如果觉得原生 SQL 麻烦可以加，也可以先裸 sqlite3 |

### 前端

| 选型 | 说明 |
|------|------|
| **HTML + CSS + JS** | 纯静态，无框架，基于 prototype.html 演进 |
| **Fetch API** | 调后端 REST API，替代 prototype 中的模拟数据 |
| **无构建工具** | 不需要 npm/webpack/vite，浏览器直接打开 HTML |

### API 风格

```
RESTful API，JSON 请求/响应
前后端分离，前端通过 fetch 调用
```

## 开发节奏

### Day 1：跑通骨架

```
目标：后端 API 跑通，前端页面能展示真实数据

□ 初始化 Flask 项目
□ 建 SQLite 数据库（6 张表）
□ 实现素材 CRUD API（POST/GET /api/material）
□ 实现拆解 CRUD API（POST/GET /api/dismantle）
□ 前端：把 prototype.html 的模拟数据替换为真实 API 调用
□ 验证：能在页面上录入素材、查看素材列表
```

### Day 2：跑通裂变

```
目标：完整闭环可演示

□ 实现骨架 CRUD API
□ 实现裂变 API（POST /api/fission）
□ 实现效果数据回写 API
□ 前端：裂变页面联调
□ 前端：骨架库页面联调
□ 验证：录入→拆解→骨架→裂变→新素材，全流程跑通
```

## 升级路径（后续迭代时考虑）

```
当前（Demo）                →  MVP                    →  正式产品
─────────────────────────────────────────────────────────────────
Flask + SQLite              →  FastAPI + MySQL       →  FastAPI + MySQL
纯 HTML/JS                  →  纯 HTML/JS             →  Vue3 全家桶
单机本地运行                →  Docker 部署             →  CI/CD + 云部署
无鉴权                      →  JWT 鉴权               →  RBAC 权限体系
手动录入                    →  CSV 批量导入            →  数据接入管道
规则匹配                    →  规则 + 效果排序         →  AI 辅助拆解/裂变
```

**关键原则：每一阶段都是上一阶段的渐进增强，不是推翻重来。**

---

## 不选的方案 & 原因

| 方案 | 不选原因 |
|------|---------|
| **Node.js + Express + MySQL** | 你主 Python，Node 开发效率不如 Python；MySQL 在 Demo 阶段过重 |
| **Vue3 全家桶** | 引入构建工具链（npm/vite/webpack），增加复杂度，Demo 阶段不值得 |
| **Spring Boot** | Java 生态，与你的技术背景和 AI 接入方向都不匹配 |
| **PostgreSQL** | 比 SQLite 重，Demo 阶段用不到它的额外特性；后续可无缝切换 |
